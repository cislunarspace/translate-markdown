# 上下文

本仓库用于开发一个将英文 Markdown 文档翻译成中文的工具。目标用户为开发者本人，运行环境为本地桌面。

## 领域词汇表

### 文档

- **源文档（Source Document）**
  待翻译的英文 Markdown 文件，扩展名为 `.md`。其所在目录同时用于存放图片等被引用的本地资源。

- **目标文档（Target Document）**
  翻译完成后生成的中文 Markdown 文件。命名规则为在源文档主文件名后追加 `_zh`，扩展名保持 `.md`，并与源文档位于同一目录。目标文档在所有翻译单元均被处理后生成。系统保证流程完整性，不保证翻译质量——翻译质量由 LLM 和 prompt 决定。

- **中间结果（Intermediate Result）**
  翻译过程中已完成的译文与待处理的剩余内容。当翻译中断时，系统应能保存中间结果，并在下次运行时从断点继续。

- **进度缓存（Progress Cache）**
  保存中间结果的隐藏文件，位于目标文档同一目录。用于在翻译中断后恢复进度。以源文档整体 hash 为键；hash 变化则全部失效，重新翻译。不做细粒度增量更新，因为段落增删会导致 unit 编号偏移，逐 unit 校验的成本和风险高于重译。

### 内容块

- **保护块（Protected Block）**
  Markdown 中不需要翻译、必须原样保留的片段。包括：
  - 代码块（fenced code blocks）
  - 行内代码
  - 图片的文件路径

- **整体可译块（Whole Translatable Block）**
  作为整体发送给 LLM 翻译的 Markdown 元素。单元格内的普通文本可译，但表格语法（管道符、分隔行）本身不能错位。当前仅 Markdown 表格属于此类。

- **可译块（Translatable Block）**
  需要由大语言模型翻译成中文的文本片段。包括普通段落、标题、列表项、图片的 alt 文本等。

- **翻译单元（Translation Unit）**
  发送给大语言模型的一次翻译请求所携带的内容。由预处理阶段对源文档进行分割产生，分割规则以物理行为主，保护块和整体可译块作为整体不拆分。

## 配置与输入

- **本地配置（Local Configuration）**
  存储在本地文件中的持久化设置，至少包含大语言模型的 API Key、请求地址与并行数量。应用启动时自动读取，用户可在 GUI 中修改。

- **并行数量（Parallel Count）**
  同时向大语言模型发送的翻译请求数量。取值范围 1~10，默认值为 3。取 1 时等价于串行翻译；大于 1 时，多个翻译单元可并发执行，但任一单元失败将停止整个翻译任务。

- **输入方式（Input Methods）**
  用户可以通过 GUI 文件对话框选择源文档，也可以通过命令行参数直接传入源文档路径。

## 实时反馈

- **实时反馈（Real-time Feedback）**
  翻译过程中向前端持续汇报的状态信息，至少包括：
  - 整体进度（百分比或已完成/总数）
  - 当前正在处理的章节或行号
  - 日志记录
  - 每个翻译单元完成后的原文与译文（GUI 中实时展示；CLI 中通过 `--verbose` 启用。单条原文或译文超过 300 字符时可截断）

## 核心约束

- 源文档与目标文档必须位于同一目录，以保证图片引用路径有效。
- 保护块不得被翻译或破坏。
- 翻译过程支持断点续翻。
- 默认假设单个翻译单元不会超过大语言模型的上下文长度。
- 应用以单文件脚本形式存在，依赖通过配置文件管理。

## 边界行为

- 表格中的代码片段作为代码保留，不翻译。
- 图片 alt 文本可译，图片路径保持不变。
- 翻译中断后，已完成的中间结果应被保存，后续可恢复。
