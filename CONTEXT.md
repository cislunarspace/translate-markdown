# 上下文

本仓库用于开发一个将英文 Markdown 文档翻译成中文的工具。目标用户为开发者本人，运行环境为本地桌面。

## 领域词汇表

### 文档

- **源文档（Source Document）**
  待翻译的英文 Markdown 文件，扩展名为 `.md`。其所在目录同时用于存放图片等被引用的本地资源。

- **目标文档（Target Document）**
  翻译完成后生成的中文 Markdown 文件。命名规则为在源文档主文件名后追加 `_zh`，扩展名保持 `.md`，并与源文档位于同一目录。

- **中间结果（Intermediate Result）**
  翻译过程中已完成的译文与待处理的剩余内容。当翻译中断时，系统应能保存中间结果，并在下次运行时从断点继续。

- **进度缓存（Progress Cache）**
  保存中间结果的隐藏文件，位于目标文档同一目录。用于在翻译中断后恢复进度。

### 内容块

- **保护块（Protected Block）**
  Markdown 中不需要翻译、必须原样保留的片段。至少包括：
  - 代码块（fenced code blocks 与行内代码）
  - Markdown 表格整体结构（单元格内的普通文本可译，但表格语法本身不能错位）
  - 图片的文件路径

- **可译块（Translatable Block）**
  需要由大语言模型翻译成中文的文本片段。包括普通段落、标题、列表项、图片的 alt 文本等。

- **翻译单元（Translation Unit）**
  发送给大语言模型的一次翻译请求所携带的内容。通常按物理行组织，但表格、代码块等保护块应作为整体（逻辑块）处理，避免拆散。

### 架构

- **前端（Frontend）**
  基于 PyQt6 构建的图形界面，用于选择源文档、配置参数、实时展示翻译进度与状态。

- **后端（Backend）**
  负责 Markdown 预处理、翻译流程编排，并调用大语言模型模块完成翻译。

- **大语言模型模块（LLM Module）**
  后端的一部分，负责与 DeepSeek API 通信。运行时需要用户提供 API Key 与请求地址。

## 配置与输入

- **本地配置（Local Configuration）**
  存储在本地文件中的持久化设置，至少包含大语言模型的 API Key 与请求地址。应用启动时自动读取，用户可在 GUI 中修改。

- **输入方式（Input Methods）**
  用户可以通过 GUI 文件对话框选择源文档，也可以通过命令行参数直接传入源文档路径。

## 实时反馈

- **实时反馈（Real-time Feedback）**
  翻译过程中向前端持续汇报的状态信息，至少包括：
  - 整体进度（百分比或已完成/总数）
  - 当前正在处理的章节或行号
  - 日志记录

## 核心约束

- 源文档与目标文档必须位于同一目录，以保证图片引用路径有效。
- 保护块不得被翻译或破坏。
- 翻译过程支持断点续翻。
- 默认假设单个翻译单元不会超过大语言模型的上下文长度。
- 应用以单文件脚本形式存在，依赖通过配置文件管理。

## 边界行为

- 表格中的代码片段作为代码保留，不翻译。
- 图片 alt 文本可译，图片路径保持不变。
- 翻译中断后，已完成的中间结果应被保存，后续可恢复。
